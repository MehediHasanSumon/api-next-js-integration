Voice Clip Implementation Steps (Next.js + Laravel)

Goal
- Allow users to record and send voice clips in chat.
- Store clips in Laravel (public storage), return attachment metadata.
- Render audio messages with a compact player in chat UI.

Step 1: Backend DB/Models
1) Confirm attachments table supports audio metadata.
   - We already store: attachment_type, storage_path, mime_type, extension, size_bytes, duration_ms, etc.
2) If duration_ms not stored for audio, ensure it is saved (we already have duration_ms column).

Step 2: Backend Validation (AttachmentController)
1) Add audio mime types to allowed list:
   - audio/mpeg (mp3), audio/wav, audio/webm, audio/ogg, audio/mp4 (m4a)
2) Keep max size (e.g., 10MB) or increase if needed.

Step 3: Backend Upload Metadata
1) When an audio file uploads, set:
   - attachment_type: "audio"
   - mime_type: detected
   - size_bytes: file size
   - duration_ms: parse if possible (optional; can be provided from client)
2) If you want exact duration on server, use a library (e.g., getID3) later. For now, allow client to send duration_ms in metadata.

Step 4: Frontend Recording UI
1) Add a mic button in chat composer.
2) Use MediaRecorder API to record:
   - On click: request mic permission, start recording.
   - On stop: create Blob (audio/webm or audio/ogg), create File object.
3) Show a small recording state:
   - Timer, cancel, send button.

Step 5: Frontend Upload Flow
1) Use existing upload endpoint:
   - POST /chat/attachments with FormData.
2) After upload, include attachment payload in sendMessage payload:
   - message_type: "audio"
   - attachments: [{ attachment_type: "audio", storage_path, mime_type, size_bytes, duration_ms }]
3) You can send with empty body.

Step 6: Frontend Rendering (MessageBubble)
1) In message list, when attachment_type === "audio":
   - Render <audio controls> with src=resolveAttachmentUrl()
   - Show filename + duration.
2) Keep image/file UI unchanged.

Step 7: Realtime & Forwarding
1) Forwarding should copy attachments (already fixed).
2) Realtime messages already include attachments.

Step 8: UX Polishing
1) Disable send while recording.
2) Show upload progress (optional).
3) Auto-stop recording on max duration (e.g., 2 mins).

Step 9: Tests
1) Backend: add test for audio attachment upload.
2) Frontend: manual QA — record, send, reload, and play audio.

Checklist
- [ ] Add audio mime types in backend validation
- [ ] Implement recorder UI + state
- [ ] Upload audio as attachment
- [ ] Send message with message_type="audio"
- [ ] Render audio player in chat UI
- [ ] Forwarding works for audio

Notes
- Start with audio/webm (Chrome/Edge) for easiest MediaRecorder support.
- For Safari/iOS, use audio/mp4 (m4a) or audio/wav. Consider fallback later.

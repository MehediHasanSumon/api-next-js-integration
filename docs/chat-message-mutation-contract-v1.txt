Chat Message Mutation Contract (V1)
Date Locked: 2026-03-06
Scope: Forward + Reaction + Remove (for you / everywhere)

1) Feature Status (Locked)
- Forward: Enabled for accepted participants only.
- Reaction: Enabled for accepted participants only.
- Remove for you: Enabled for any visible participant (accepted/pending if message is visible to that user).
- Remove from everywhere: Enabled with strict permission/time-window policy.

2) Role and Permission Policy (Locked)
- Sender:
  - Can forward any message visible to sender to conversations where sender is accepted participant.
  - Can react on messages in accepted conversations.
  - Can remove own message from everywhere within configured time window.
  - Can always remove any visible message for self.
- Receiver (non-sender participant):
  - Can forward visible messages only if receiver is accepted in target conversation.
  - Can react in accepted conversations only.
  - Cannot remove others' messages from everywhere.
  - Can remove visible messages for self.
- Admin (moderation):
  - Can remove any message from everywhere at any time.
  - Can remove for self as normal.
  - Forward/reaction follow accepted-participant requirement in conversation context.

3) Remove-from-Everywhere Window and Exceptions (Locked)
- Default window: 15 minutes from `messages.created_at`.
- Sender can remove-from-everywhere only inside 15-minute window.
- Admin bypasses time window (anytime).
- Exceptions:
  - If message already removed-from-everywhere, repeat request is idempotent (success/no-op).
  - System messages are not removable by normal users; admin-only policy applies.
  - If conversation access is lost, operation is forbidden (403).
  - If message not found or not visible, return 404.

4) Tombstone and Visibility Behavior (Locked)
- Remove-from-everywhere does not physically delete row immediately; convert to tombstone state.
- Tombstone display text: "This message was removed."
- Tombstone message cannot be edited/reacted/forwarded by normal users.
- Remove-for-you hides message only for actor user and must sync across actor's devices.

5) Operational Constants (Locked)
- `REMOVE_EVERYWHERE_WINDOW_MINUTES = 15`
- `REMOVE_EVERYWHERE_ADMIN_BYPASS = true`
- `MUTATION_APPLIES_TO_ACCEPTED_ONLY = true` (except remove-for-you on visible messages)

6) UX Behavior (Locked)
- Forwarded label format:
  - Primary label: `Forwarded`
  - If original sender is known: show secondary text `from {name}`.
  - If unknown/removed: fallback `from Unknown`.
- Reaction UI style:
  - Inline reaction chips below each message bubble.
  - Quick picker opens from message action menu (desktop click, mobile long-press action).
  - Toggle behavior: clicking same emoji by same user removes that reaction.
  - Users can apply multiple different emojis on same message.
- Deleted message display:
  - Remove from everywhere tombstone text: `This message was removed.`
  - Tombstone keeps message timestamp and sender alignment, but disables reply/forward/react actions.
  - Remove for you: message is hidden from actor UI completely (no tombstone shown to actor).

7) Query Scope Contract (Locked)
- Message visibility scope: `Message::visibleToUser($userId)`
  - Excludes rows hidden for user via `message_receipts.hidden_at`.
  - Keeps row visible for all other participants.
- Reaction aggregate scope: `Message::withActiveReactionAggregates($viewerUserId)`
  - Includes `reactions_total`.
  - Includes grouped emoji aggregates (`emoji`, `count`, `reacted_by_me`).
- Reuse policy:
  - Message list API uses both scopes.
  - Conversation list/show last-message loading uses both scopes.

8) Forward API Contract (Locked)
- Endpoint: `POST /api/chat/messages/{message}/forward`
- Required payload:
  - `target_conversation_id` (must exist)
- Optional payload:
  - `body` or `comment` (forward note)
  - `metadata`
  - `client_uid`
- Validation and access:
  - Source message must be visible to actor.
  - Actor must be accepted participant in target conversation.
- Persistence:
  - `forwarded_from_message_id` and `forwarded_from_user_id` are populated.
  - Immutable `forwarded_snapshot` is stored at forward time.

9) Reaction Toggle API Contract (Locked)
- Endpoint: `POST /api/chat/messages/{message}/reactions`
- Payload:
  - `emoji` (required)
- Validation and access:
  - `emoji` must be emoji-like content.
  - Actor must be accepted participant in the message conversation.
  - Message must be visible to actor.
- Behavior:
  - If same (`message_id`,`user_id`,`emoji`) exists => remove (toggle off).
  - If not exists => create (toggle on).
- Response includes:
  - `action` (`added` or `removed`)
  - `reactions_total`
  - grouped `reaction_aggregates` with `emoji`, `count`, `reacted_by_me`

10) Reaction Remove API Contract (Locked)
- Endpoint: `DELETE /api/chat/messages/{message}/reactions`
- Payload:
  - `emoji` (required)
- Validation and access:
  - Same access rules as toggle endpoint (message visible + accepted participant).
- Behavior:
  - Explicitly removes only actor's selected emoji reaction from message.
  - Idempotent: if reaction does not exist, still returns success with latest aggregates.
- Response shape:
  - Same as toggle endpoint to support thread live patching:
    - `conversation_id`
    - `message_id`
    - `emoji`
    - `action` (`removed`)
    - `reactions_total`
    - `reaction_aggregates`

11) Remove For You API Contract (Locked)
- Endpoint: `POST /api/chat/messages/{message}/remove-for-you`
- Access:
  - Actor must be a visible participant of the message conversation.
  - Accepted/pending participant allowed for self-hide behavior.
- Behavior:
  - Writes/updates `message_receipts.hidden_at` for authenticated user only.
  - Does not delete row from `messages` table.
  - Idempotent for same user + message.
- Sync contract:
  - Emits `chat.message.removed` event on `private(user.{actorId})` with mode `for_you`.
  - Supports same-user multi-device synchronization.
- Response includes:
  - `conversation_id`
  - `message_id`
  - `mode` (`for_you`)
  - `actor_user_id`
  - `removed_at`

12) Remove From Everywhere API Contract (Locked)
- Endpoint: `POST /api/chat/messages/{message}/remove-for-everywhere`
- Access:
  - Actor must be visible participant in conversation.
- Policy:
  - Admin can remove any message for everyone anytime.
  - Non-admin must be message owner.
  - Non-admin owner allowed only within `REMOVE_EVERYWHERE_WINDOW_MINUTES` (15 min).
  - `system` messages are admin-only for remove-everywhere.
- Tombstone conversion:
  - `message_type` => `system`
  - `body` => `This message was removed.`
  - `metadata` stores removal actor/time/original fields.
  - attachments deleted from `message_attachments`.
  - `messages` row remains intact.
- Sync contract:
  - Emits `chat.message.removed` on `private(conversation.{conversationId})` with mode `everywhere`.
- Response includes:
  - `conversation_id`, `message_id`, `mode`, `actor_user_id`, `removed_at`, `message`

13) Message Mutation Service Contract (Locked)
- Service: `App\Services\Chat\MessageMutationService`
- Responsibilities:
  - Centralize forward/reaction/remove mutation orchestration.
  - Resolve visible message and participant access checks consistently.
  - Keep controller methods thin and uniform.
- Controller rule:
  - `MessageController` delegates mutation workflows to mutation service.
  - Controller keeps only request/response mapping.
- Transaction rule:
  - Destructive writes remain wrapped in DB transactions in `ChatMessagingService` mutation methods.

14) Validation and Error Mapping Contract (Locked)
- FormRequest usage:
  - `ForwardMessageRequest`
  - `ToggleMessageReactionRequest`
  - `RemoveMessageReactionRequest`
  - `RemoveMessageForYouRequest`
  - `RemoveMessageForEverywhereRequest`
- Standard API error payload mapping:
  - 401 => `UNAUTHENTICATED`
  - 403 => `FORBIDDEN`
  - 404 => `NOT_FOUND`
  - 409 => `CONFLICT`
  - 422 => `VALIDATION_ERROR` (+ `errors` bag)
- Specific chat access messages:
  - pending => `Conversation request is pending. Accept it before this action.`
  - archived => `Conversation is archived. Unarchive it before this action.`
  - generic forbidden => `You are not allowed to perform this action in this conversation.`

15) Forward Realtime Contract (Locked)
- Forwarded message publish path:
  - Event class: `MessageSent`
  - Channel: `private(conversation.{targetConversationId})`
  - Event name: `chat.message.sent`
- Payload requirements:
  - `event_type` => `forwarded`
  - `message` includes forwarded columns and `client_uid`
  - `forward` metadata block includes:
    - `forwarded_from_message_id`
    - `forwarded_from_user_id`
    - `forwarded_snapshot`
    - `client_uid`
- Dedupe rule:
  - `client_uid` is always populated for forwarded messages (request-provided or server-generated fallback).

16) Reaction Realtime Contract (Locked)
- Reaction update publish path:
  - Event class: `MessageReactionUpdated`
  - Channel: `private(conversation.{conversationId})`
  - Event name: `chat.message.reaction.updated`
- Payload requirements:
  - `conversation_id`
  - `message_id`
  - `emoji`
  - `action` (`added` or `removed`)
  - `user_id`
  - `reactions_total`
  - `reaction_aggregates` (`emoji`, `count`, `reacted_by_me`)
  - `sent_at`
- Channel policy:
  - Broadcast target is conversation private channel only.
  - Delivery is limited by `conversation.{conversationId}` authorization (visible participants only).

17) Message Remove Realtime Contract (Locked)
- Message remove publish path:
  - Event classes: `MessageRemovedForUser`, `MessageRemovedEverywhere`
  - Event name: `chat.message.removed`
- For-you mode contract:
  - Channel: `private(user.{actorUserId})`
  - `mode` payload value: `for_you`
  - Purpose: cross-device sync for same authenticated user.
- Everywhere mode contract:
  - Channel: `private(conversation.{conversationId})`
  - `mode` payload value: `everywhere`
  - Purpose: sync tombstone/remove state for all visible conversation participants.
- Shared payload keys:
  - `conversation_id`
  - `message_id`
  - `mode`
  - `actor_user_id`
  - `removed_at`

18) Conversation List Correctness Contract (Locked)
- Last message preview correctness:
  - Conversation list resolves the latest visible message per viewer from `messages` table.
  - It no longer depends only on `conversations.last_message_id` when that row is hidden/deleted for viewer.
  - `last_message_at` in list response mirrors viewer-visible latest message timestamp.
- Forward mutation correctness:
  - Forwarded message in target conversation is used as latest preview immediately after mutation.
- Hidden latest message correctness:
  - If viewer hides the latest message (`remove-for-you`), preview falls back to previous visible message.
- Unread count correctness:
  - `remove-for-you` triggers unread count recomputation for actor participant.
  - Hidden messages are excluded from unread counting, including when hidden row was latest unread.
- Filter stability:
  - `inbox`, `requests`, and `archived` filters continue to use participant state flags.
  - Remove/forward mutations do not alter `participant_state` or `archived_at` semantics.
